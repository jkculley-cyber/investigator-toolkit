## What Was Done This Session

### Mobile Layout — Hamburger Drawer + Responsive Shell

Implemented full mobile responsiveness for the app layout. No individual page files were touched — all changes are in the layout layer and apply universally.

#### Files Changed

| File | Change |
|------|--------|
| `src/contexts/SidebarContext.jsx` | NEW — `sidebarOpen` / `setSidebarOpen` state context |
| `src/components/layout/AppShell.jsx` | Wrapped with `SidebarProvider`; split into inner component; mobile backdrop overlay (tap to close) |
| `src/components/layout/Sidebar.jsx` | Fixed overlay drawer on mobile (slide in/out via transform); always visible on desktop (`md:relative`); X close button (mobile only); all NavLinks call `setSidebarOpen(false)` on tap |
| `src/components/layout/Topbar.jsx` | Hamburger button (mobile only, left of title); responsive padding `px-3 md:px-6` |
| `src/pages/LoginPage.jsx` | Responsive logo `h-28 sm:h-36 md:h-44`; card padding `p-5 sm:p-8` |
| `CLAUDE.md` | Corrected hosting from "Netlify" to "Cloudflare Pages (via GitHub Actions)" |

#### How It Works
- **Mobile (< 768px):** Sidebar hidden by default. Hamburger in Topbar opens it. Dark backdrop behind open drawer — tap backdrop or any nav link to close.
- **Desktop (≥ 768px):** Sidebar always visible, exactly as before. Hamburger hidden. No behavior change.
- State shared via `SidebarContext` — no prop drilling through page files.
